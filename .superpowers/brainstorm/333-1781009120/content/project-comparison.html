Desk-Pet vs Sakura - 项目对比总览

两个桌宠项目的技术架构和功能差异

Desk-Pet (本项目)

技术栈: Electron 42 + HTML/CSS/JS

角色: Live2D (Hiyori 模型)

窗口模式: 三模式 (宠物/壁纸/软件)

亮点:

  • Live2D 动态角色 + 物理模拟
  • WebGL 壁纸引擎
  • Wallpaper Engine .pkg 导入
  • 番茄钟 + RPG 任务系统
  • 关系/世界/故事进度系统

缺失/粗糙:

  • 无设置界面 (API key 需手动改 JSON)
  • 主动交互是空壳 (proactive-trigger stub)
  • 点击穿透未真正实现
  • 无 TTS / 记忆系统
  • 动画控制器未接入
  • Shader 特效未使用
  • 无玻璃拟态等视觉打磨
Sakura (参考项目)

技术栈: Python 3.14 + PySide6/Qt

角色: 静态 PNG + 交叉淡入淡出

窗口模式: 单一透明置顶窗口

亮点:

  • 完整 Agent 工具循环 (10步/轮)
  • 打字机字幕 + 毛玻璃输入框
  • 主动关怀 (定时截图 + LLM判断)
  • 长期记忆 (mem0 + Qdrant 向量库)
  • 多 TTS 引擎 + 预生成管线
  • 完整设置界面 (9个选项卡)
  • 11色调色板主题系统
  • 提醒系统 + 桌面控制 (MCP)
  • 插件系统 + 角色包导入导出

核心差距分析

Desk-Pet 优势 (应保留强化)

  • Live2D 动态角色 (比静态PNG更生动)
  • WebGL 壁纸渲染引擎
  • 番茄钟 + RPG 任务系统
  • 关系/章节/世界进度系统
  • 三模式架构灵活性

可从 Sakura 借鉴的改进

  • 打字机字幕效果 → 替换现有的静态气泡
  • 毛玻璃输入框 → 提升UI质感
  • 设置界面 → 让用户可视化配置
  • 主动交互系统 → 从 stub 变为真正实现
  • 长期记忆系统 → 增强对话连贯性
  • 表情切换动画 → Live2D motion + 过渡
  • 主题系统 → 可配置的配色方案
