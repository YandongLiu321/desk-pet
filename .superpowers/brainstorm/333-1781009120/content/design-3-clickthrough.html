Step 2: 真正的点击穿透

pet 窗口默认 click-through，仅角色和交互区域响应鼠标

当前问题

整个 500x500 的 pet 窗口对鼠标完全不透明——桌面图标被遮挡且无法点击。设计文档中提到了 pointer-events: none 方案但从未实现。

方案

默认状态（click-through）

body { pointer-events: none; }

整个窗口对鼠标透明

仅角色区域：.character-area { pointer-events: auto; }

角色可拖拽移动窗口

右键弹出菜单

单击打开对话面板

交互状态（面板展开时）

body.interactive { pointer-events: auto; }

展开对话面板 / 任务面板时

整个窗口恢复鼠标响应

输入框、按钮可正常使用

ESC 或点击外部 → 移除 .interactive → 恢复穿透

实现要点

  • CSS: body { pointer-events: none; user-select: none; }
  • 角色区域 + 菜单 + 面板容器: .character-area, .context-menu, .floating-panel { pointer-events: auto; }
  • JS: document.body.classList.add('interactive') 切换全局交互
  • 面板关闭时自动移除 .interactive
  • Electron 窗口: setIgnoreMouseEvents(true, { forward: true }) 作为底层保障
  • Main process 通过 IPC 提供 window:setIgnoreMouseEvents 支持精细化控制

交互区域示意

角色区域 (pointer-events: auto)
其余区域: pointer-events: none
